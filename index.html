Projects

Proxmox Server

I set up a proxmox server on a used laptop. From there, I set up an instance of Windows Server 2022 and a Kali Linux instance. I set up my server to give DHCP and DNS services to my machines on proxmox. From there, I got an IP address and DNS services were given by the server rather than my own WiFi router.

While playing around with my proxmox server, I wanted to see if I could access my server from outside of my LAN. I thought about port forwarding, but port forwarding is insecure and could open my LAN to attacks. So I also installed Tailscale onto my instance of proxmox, so now I can access my Proxmox machine from anywhere as long as I am connected to Tailscale from the machine that I am using outside of the LAN.

Amazon Web Services

I did the same project as with the proxmox project on the left, except I decided to use AWS to attempt to replicate my work. For that, I created my own Virtual Private Cloud, and created a public and private subnet. From there, I put my Windows server in my public cloud and an instance of Amazon Linux in my private instance and connected to it through another Linux instance in my public subnet. I did not use Windows Server's DHCP service, as AWS has its own, but I was able to have it run its DNS service, and tested it on my private Linux instance.

Part of my learning process here was to learn how AWS's VPC and EC4 services worked and how they related to each other. I also had to learn how to give my public subnet a route table and an internet gateway to get the cloud to access the internet and vice versa. I made the route table for the public subnet point to the internet. For my private subnet, I had to create another route table, but instead of pointing it to the internet, I pointed it to a NAT gateway, so that the internet cannot access the private subnet, and therefore, my private Linux instance.

Self-hosted Network Attached Storage

Through my CompTIA Network+ exam, I learned about what a Network Attached Storage (NAS) was, and I wanted to see if I could replicate my own rather than using a cloud storage like Google Drive or OneDrive.

Using a Raspberry Pi 5, I downloaded OpenMediaVault and attached a hard drive to my machine. From there, I set up an admin and user account so I could access my storage. I also used Tailscale as a means to connect to it using my computer's Server Message Block (SMB) protocol to allow access to it from outside of my LAN.
